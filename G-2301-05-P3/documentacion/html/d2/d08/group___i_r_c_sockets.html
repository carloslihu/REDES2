sockets
tools

Funciones que se encargan de manejar sockets

openSocket_TCP

abre un socket TCP
Returns
el socket o -1 en caso de que hubiera un fallo

bindSocket_TCP

enlaza un socket TCP con un puerto a la vez que rellena los campos de la estructura sockaddr_in de serv_addr
Parameters
sockfdel socket a enlazar
portnoel puerto al que enlazar el socket
serv_addrla dirección de una estructura sockaddr_in en donde se guardarán los valores relativos a esta conexión
Returns
-1 en caso de error, un numero positivo en otro caso

acceptConnection

prepara un socket para aceptar una conexión entrante
Parameters
sockfdel socket que está escuchando conexiones entrantes
cli_addrla dirección de una estructura sockaddr_in donde se rellenaran los valores relativos al otro extremo de la conexión
Returns
-1 en caso de error y un nuevo socket (valor mayor que 0) en el que se mantendrá la comunicación con el otro extremo

connectTo

trata de conectarse a un puerto de un host
Parameters
sockfdel socket que intentará realizar la conexión
hostnameel nombre de la maquina a la que se desea conectarse
portnoel puerto al que se desea conectar
Returns
en caso de error, un valor negativo. De lo contrario, un nuevo socket en el que se mantendrá la comunicación con el otro extremo

connectToIP

trata de conectarse a un puerto de una IP
Parameters
sockfdel socket que intentará realizar la conexión
IPla dirección IP en notación estándar a la que conectarse
portnoel puerto al que se desea conectar
Returns
en caso de error, un valor negativo. De lo contrario, un nuevo socket en el que se mantendrá la comunicación con el otro extremo

openSocket_UDP

abre un socket UDP
Returns
el socket o -1 en caso de que hubiera un fallo

bindSocket_UDP

enlaza un socket UDP con un puerto a la vez que rellena los campos de la estructura sockaddr_in de serv_addr
Parameters
sockfdel socket a enlazar
portnoel puerto al que enlazar el socket
serv_addrla dirección de una estructura sockaddr_in en donde se guardarán los valores relativos a esta conexión
Returns
-1 en caso de error, un numero positivo en otro caso

getSocketPort

obtiene el puerto al que ha sido enlazado el socket sockfd y tambien rellena los campos de serv
Parameters
sockfdel socket (enlazado) del que queremos sacar su puerto
servla estructura sockaddr_in en la que se guardan los datos referentes a nuestro extremo de la conexion
Returns
el puerto del socket. Si hubo un error, -1

iniAddrUDP

inicializa correctamente la estructura con datos para una conexion UDP
Parameters
si_otherla estructura que se inicializara
portel puerto del otro sistema con el que se desea comunicar
hostnameel hostname del otro sistema con el que se desea comunicar
Returns
-1 si hubo un error, IRC_OK si todo fue bien
